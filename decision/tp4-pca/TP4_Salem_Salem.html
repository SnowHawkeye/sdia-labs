2. Principal Component Analysis¶

The purpose of this part is to observe the respective contributions of each component of a PCA of images of 5. The function sklearn.decomposition.PCA of scikit-learn is available. In practice, one must first estimate the mean vector and then work with centered data.

Documentation¶

First have a look at http://scikit-learn.org/stable/modules/generated/sklearn.decomposition.PCA.html

In [ ]:
# Principal component analysis
moy_train = x_train_brut.mean(axis=0)  # all the data, 5 & 6
x_train_centre = x_train_brut-np.tile(moy_train,(N_train,1))

# PCA from scikit-learn
from sklearn.decomposition import PCA
pca = PCA()
pca.fit(x_train_centre)  # you may forget centering that is done by sklearn PCA

singval = pca.singular_values_   # eigenvalues
comp = pca.components_           # principal components
proj = pca.transform(x_train_centre)  # computes the projection coefficients

Question 1¶

  • PCA takes as input the number of components to keep. In the example above, this parameter was not specified and all the components of the decomposition were therefore kept.
  • The data used to determine which components are the most important is given to the fit function.
  • Once the PCA model is fit to the data, we can retrieve the directions of the projection with the components_ attribute. These directions are the eigenvectors of the matrix obtained by singular value decomposition. The corresponding eigenvalues can be obtained with the eigenvalues attribute, even though they're not useful in this example.
  • To project an image in the new basis, we can compute the coefficients thanks to the transform function.

Question 3¶

With the number above, we can distinguish a 5 with only 2 components. It seems very satisfactory with 5 components, although there is noise around the number (which would normally be black pixels). It shows the interest of using PCA: we divide by $2^7$ the number of components needed to recognize a 5 - and therefore the computational charge - which is quite impressive. Obviously this comes at the cost of accuracy when reconstructing the image, which is not a problem for every use case (cf. Exercise 2).

Here, with more than 2 components the "noise" makes the reconstruction very blurry. Once again, we seem to have a nice result with only 2 components. The conclusion we can draw from this is that PCA is good at extracting the most significant dimensions, but that an image reconstructed with a lot of components is not necessarily close to the original. A reason for that can be that PCA does not take into account the differences between classes, and we have here a dataset with both 5's and 6's. As a consequence, the "noise" that we mentioned could be due to the other classe's influence (overlap between classes).

Questions 2 & 3¶

The data seems almost linearly separable. As a consequence, we suppose that using LDA would also provide good results. As we demonstrated in TP3, LDA is sensitive to outliers, but there does not seem to be too many of them in this training set, which supports our proposition.

To demonstrate this, we will use sklearn's LDA implementation. For representation purposes, instead of using the actual training dataset we will use the 2D projections. We will also use a utility class we made in TP3 to display the decision boundary.

In [ ]:
from sklearn.discriminant_analysis import LinearDiscriminantAnalysis
from tp4_annex import BinaryPointsClassificationPresenter # utility class made for TP3

test_data_lda = np.hstack([np.ones((345,1)), proj[:, 0:2]]) # we need a column of ones for BinaryPointsClassificationPresenter
class_train = np.ones((345,)) # label 1 for digit 6
class_train[:145] = 0 # label 0 for digit 5

lda = LinearDiscriminantAnalysis()
lda.fit(test_data_lda, class_train)

x1_bounds, x2_bounds = (-8,11), (-9,10)
bpc = BinaryPointsClassificationPresenter(test_data_lda, class_train, lda.predict)
bpc.display_prediction(x1_bounds, x2_bounds, title=f"Error rate : {round(bpc.error_rate*100, 2)}%")

As expected, LDA is able to find a "relevant" decision boundary between 5's and 6's. We do however recall that the LDA model was trained with 2D coordinates: it is possible that the excess data from the other coordinates would have made the training slightly better or worse.

Let's try to make the prediction with the full 256-dimension dataset. We won't be able to visually represent it but we can compare its error rate to the one obtained in 2D.

In [ ]:
from sklearn.metrics import zero_one_loss

lda = LinearDiscriminantAnalysis()
lda.fit(x_train_brut, class_train)
prediction = lda.predict(x_train_brut)
loss = zero_one_loss(class_train, prediction)
print(f"The error rate with 256 dimensions is {loss * 100} %.")

The error rate with 256 dimensions is 0.0 %.

In this case the prediction is better with 256 dimensions than with 2 dimensions, which could be expected. We can even notice that the classifier doesn't make any mistake, which shows that in 256 dimensions the data is exactly linearly separable: it confirms the conjecture that PCA had allowed us to make in 2D.

As a summary, an application of PCA is to conduct an a priori study of the dataset, which helps choosing a learning algorithm. The example above - checking if data is linearly separable - is very representative of this.
